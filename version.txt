Release Notes:
==============

version 0.2.0b:
--------------

not released yet

	- StringField: added maxLength, minLength attributes
	- Added MultiAutoComplete
	- typeInvitation for StringField and subclasses
	- MultiSelectField
	- DDList widget: sortable list widget
	- Made TimeField, DateTimeField, PairField inherit from CombineField
	- CombineField: replace PairField, but for N fields
	- Markup and CSS refactoring: the "description" and "label" attributes are now in the field JSON
	- PasswordField: Strength indicator
	- PasswordField: CapsLock validation
	- Form mask
   - small visualization/templating engine (Visus.js)
	- Added a SliderField (type: slider)
	- Added an inputEx.importJsonSchema (experimental) method to create a form from a JSON Schema
	- Documentation now uses JsDocToolkit 2 and http://code.google.com/p/neyricjslibs-template/
	- Added a DatePickerField (type: datepicker)
	- Added a Calendar Widget (for the date picker, much lighter than YUI calendar)
	- Added DateTimeField (type: datetime)
	- Added IntegerField (type: integer) and removed the "numbersOnly" option from StringField
	- Added NumberField (type: number)
	- Added TimeField	(type: time)
	- TypeField: implemented setValue, the builder can now "load" a group config
	- Added a "destroy" method on Field
	- bugfix: URL are case sensitive
	- moved the namespace from inputEx to YAHOO.inputEx (declared using YAHOO.namespace) => breaks namespace
	- enclosed components in anonymous functions with shortcuts
	- use YAHOO.lang.extend(...,...,{ methods: }) to save some bytes => breaks the API doc generation
	- moved build/inputex-all.js to build/inputex.js for YUI loader
	- TypeField/FormBuilder update: includes more options for more fields
	- Group: removed 'optional' option.  You should instead use a sub-group (that can be collapsible as before)
	- Group: wrapped in a "fieldset" tag, collapsible option (+example), 
				use 'CSS float left' instead of tables to position elements => breaks old CSS for groups
 	- UrlField: make favicon optional otherwise it breaks https
 	- ListField: set the focus to the added element
	- Added the inputEx.Group.prototype.getFieldByName method
	- Added a focus method to set the focus on inputs
	- Moved the two functions (Array.prototype compact & indexOf) into the inputEx namespace as static functions
	- VERSION added in inputEx object
	- use YAHOO.lang.isMethods instead of typeof
	- RTEField: added SimpleEditor support and "opts" option to configure the toolbar (added by Dav Glass)
		* added a SimpleEditor example on the RTEField page
